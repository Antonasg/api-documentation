  • Validator

    The core validator, used for validations that require parameters. For example, a string length validator that uses a dynamic value to evaluate by.

    public enum StringLength: Validator { case min(Int) case max(Int) case containedIn(Range)

     public func validate(input value: String) throws {
     let length = value.characters.count
     switch self {
        case .min(let m) where length >= m:
            break
        case .max(let m) where length <= m:
            break
        case .containedIn(let range) where range ~= length:
            break
        default:
            throw error(with: value)
     }
 }

    }

    And used like:

    let validated = try "string".validated(by: StringLength.min(4))

  • Validatable

    Validatable struct is used to indicate that a value can be validated. It requires no special attributes or conformance.

    It’s purpose is largely based on convenient code share and by conforming an object inherits a great deal of syntax to more conveniently work with validators.

    Naming Conventions

    • tested throws -> Self
    • passed -> Bool
    • validated throws -> Valid
